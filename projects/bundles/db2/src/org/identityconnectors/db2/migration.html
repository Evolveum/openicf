Differences between DB2ResourceAdapter and DB2Connector

Configuration properties

  • Resource adapter supported type2 CLI legacy driver connection properties.
    1. adminAcct : Admin user account
    2. password : Admin user password
    3. dbName : Local alias name
    4. jdbcDriver : Classname of type2 driver
    5. jdbcSubprotocol : protocol used to communicate with db2, default to db2
    6. removeForeignGrants : Whether we replace all grants on update

  • Connector supports datasource,type4 and type2 connection type
    • Datasource connection type properties
      1. dataSource : JNDI name of datasource
      2. dsJNDIEnv : Multivalulue jndi entries in key=value format
      3. adminAccount : Admin user name, optional
      4. adminPassword : Admin user password,optional
    • Type4 connection type properties
      1. adminAccount : Admin user name
      2. adminPassword : Admin user password
      3. jdbcDriver : Classname of type4 jdbc driver
      4. host : Hostname or IP address of DB2 server
      5. port : Port that DB2 manager listens to
      6. databaseName : Name of ocal or remote database
      7. subprotocol : db2,db2iSeries. Default to db2
    • Type2 connection type properties
      1. adminAccount : Admin user name
      2. adminPassword : Admin user password
      3. jdbcDriver : Classname of type4 jdbc driver
      4. databaseName : Name of alias local or remote alias , database is catalogized by
      5. subprotocol : db2,db2iSeries. Default to db2

Configuration properties differences summary

ResourceAdapter supported only type2 connection type, tested with DB2 CLI legacy driver. This driver is now deprecated by IBM, however still included in DB2 9x. This driver can be removed in future DB2 release. Connector supports datasource,type4,type2 connection type. IBM DB2 Driver for JDBC and SQLJ and CLI legacy driver were tested. There are small changes in names of connector configuration properties.
