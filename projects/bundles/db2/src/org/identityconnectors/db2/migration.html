Differences between DB2ResourceAdapter and DB2Connector

Configuration properties

  • Resource adapter supported type2 CLI legacy driver or Net type 3 driver connection properties. These properties were supported by adapter.
    1. adminAcct : Admin user account
    2. password : Admin user password
    3. dbName : Local alias name in case of app driver or remote/local database in case of net driver
    4. jdbcDriver : Classname of type2 driver
    5. jdbcSubprotocol : protocol used to communicate with db2, default to db2, or full instance address : db2:hotst:port
    6. removeForeignGrants : Whether we replace all grants on update

  • Connector supports datasource,type4 and type2 connection type. Connector supports these properties.
    • Datasource connection type properties
      1. dataSource : JNDI name of datasource
      2. dsJNDIEnv : Multivalulue jndi entries in key=value format
      3. adminAccount : Admin user name, optional
      4. adminPassword : Admin user password,optional
    • Type4 connection type properties
      1. adminAccount : Admin user name
      2. adminPassword : Admin user password
      3. jdbcDriver : Classname of type4 jdbc driver
      4. host : Hostname or IP address of DB2 server
      5. port : Port that DB2 manager listens to
      6. databaseName : Name of ocal or remote database
      7. subprotocol : db2,db2iSeries. Default to db2
    • Type2 connection type properties
      1. adminAccount : Admin user name
      2. adminPassword : Admin user password
      3. jdbcDriver : Classname of type4 jdbc driver
      4. databaseName : Name of alias local or remote alias , database is catalogized by
      5. subprotocol : db2,db2iSeries. Default to db2
    • replaceAllGrantsOnUpdate : Kept for backward compatibility instead of removeForeignGrants. When set to true(default) , we replace value of passed grants on update. Otherwise we do addition of passwd grants with grants use already has. This property can be removed in future version, when IDM will properly call UpdateAttributeValuesOp operations.

Configuration properties differences summary

ResourceAdapter supported type2 app driver and type3 net driver. Net driver is removed from DB2 9x and app driver is deprecated. App driver can be removed in future DB2 release. Connector supports datasource,type4,type2 connection type. IBM DB2 Driver for JDBC and SQLJ and CLI legacy driver were tested. There are small changes in names of connector configuration properties. Connector still supports removeForeignGrants property. Connector treats grants as multivalue attribute and always does replace for update operation.

Attributes differences summary

Attributes for adapter and connector are same.

Migrating resource adapter data

DB2 connector uses same DB2 system tables like DB2 resource adapter, so no migration of users or grants is needed. We just need to create new Connector pointing to the same DB2 database.
