The MySQLUser connector specification The connector capabilities are pretty limited. The connector support just a user creation from modeluser template The update is possible just for a user name and password. No other attributes are currently supported. The delete and search of user is also supported.
The MySQLUSer connector use the template user called the "modeluser". Model user and its rights are taken taken as a template for creating every new user on the MySQL resource. What kind of operations are supported by MySQLUser connector The MySQLUser connector supports create, update, delete operation for the IdM account. Also search by user name are supported. To search by SQL user rights currently is not supported. What kind of features are supported by MySQLUser connector The MySQLUser does not support any of the special features except to set a new password. No enable/disable nor password expiration are supported. The connector does not support any scripting. Which version of framework? The framework version 1.0 and above Backward compatible with adaptor The connector is backward compatible with adaptor. All features from the adapter has been implemented exept the "JDBC URL Template" property. The JDBC URL Template configuration property is not required, because all versions of MySQL supports the same URL schema. The MySQL resource description Provides implementation of connector to MySQL database resource. MySQL Connector uses MySQL database mysql.user table to manage users. There could be a multiple records for particular user. The unique key of the record is a sequence of host, user, schema, table. The rest of the record consist of encoded user password and SQL user rights.
The MySQL resource accepts the % wildcard. This wildcard can be used for specifying any host, any user, any schema, any table. The MySQL resource restrictions The user account must have rights to read, create, update and delete the users from any hosts and for any schema and table. The modeluser can have any rights. To use the same account for user and modeluser is not recommended and can leads to problems.
The MySQL connector does not difference between the hosts. Every access rights is considered as valid from any hosts. It must imply the default wildcard "%" for hosts. It is not recommended to set up the model user with different SQL access rights which differs for hosts. See {@link org.identityconnectors.mysqluser.MySQLUserConfiguration} and {@link org.identityconnectors.mysqluser.MySQLUserConnector} for more information about DB2 connector.
