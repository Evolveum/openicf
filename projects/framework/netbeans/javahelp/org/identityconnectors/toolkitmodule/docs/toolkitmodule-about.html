Overview

The IdentityConnectors Toolkit is built to help drive development of Connectors. Connectors provide a consistent generic layer between applications and target resources. The main focus of the API is provisioning operations and password management. The toolkit is intended to facilitate development with as few dependences as possible.

Toolkit Contents

The directory where you extracted this toolkit contains the following elements:

  • The Connectors Framework
  • Full API & SPI Documentation
  • The Connector Contract Test Suite
  • A Connector Creation Wizard
  • A Groovy Console for testing and debugging

The Netbeans Plugin

*Notice: Netbeans uses its own (built-in) copy of Ant, so if you intend to use Netbeans to build and test your connectors, you need to copy junit4.jar to the IDE's Ant/lib directory as well. For Netbeans, the Ant distro is found at ${netbeans-home}/java2/ant. See for more information.

This plugin is a visual wrapper for the Ant-based Connector Creation Wizard that is provided with the toolkit. It will create and integrate skeleton connectors into the Netbeans environment based on your input. Usage is simple:

Step 1: Set Properties

  1. Tell Netbeans where you extracted the Connectors Toolkit.
  2. Choose a directory in which to create your new connector.
  3. Enter the name of the target resource.
  4. Enter a package name for the generated classes (or use the default).

Step 2: Choose SPI Operations

All the SPI operations (authenticate, create, delete, etc.) are opt-in, so you may choose whichever ones your resource supports. When finished, click "Finish", and Netbeans will generate a skeleton connector implementing the interfaces you selected. It will also set up the necessary library and javadoc references as well. The new connector project will be automatically opened and at this point is ready for you to begin working on.
