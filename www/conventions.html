Coding Conventions

The Identity Connectors project follows the Java standard coding conventions. They include standards for filenames, file organization, indentation, comments, declarations, statements, white space, naming conventions, and general programming practices. Code examples are included.

Commit rules

These rules should be followed with the best possible effort:

  • commit only buildable code
  • all tests should pass (if you need commit anyway use @Ignore)
  • do NOT commit proprietary files
  • do NOT commit build artifacts
  • use "Issue #<number> comment" template
  • propagate changes to downstream projects
