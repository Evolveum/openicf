Using IdentityConnectors with Sun™ Identity Manager (IDM)

IDM 8.1 contains support for using Identity Connectors to manage resources. Identity Connectors are a new open-source equivalent of IDM resource adapters which provide much better separation of between the app and its resources. Identity Connectors are available for download in both source code and binary formats (a .jar file referred to as a bundle) from the downloads section.

Why would I want to use Identity Connectors?

  • Open-source
  • Improved upgrade support (vs. resource adapters)
    • No IDM upgrade necessary to use newer versions of Identity Connectors
  • Classpath isolation allows support for using multiple versions of a resource's API from a single IDM instance

Integration with existing Resource Adapters

IDM 8.1 can simultaneously support both resource adapters and Identity Connectors. Your existing resource adapters will continue to work in IDM 8.1 the same as in previous releases. Once an Identity Connector is properly installed, you will create Resources using them just as you do currently with resource adapters. All other IDM tools (forms/workflows) are based upon Resources, and thus are not affected by the choice of an underlying Identity Connector versus resource adapter.

How to Install a Connector

IDM searches for local Identity Connector bundle files with .jar suffix in the WEB-INF/bundles directory within your web app. First, make sure that this folder is present in your IDM deployment. If it doesn't exist, create it.

Next, before deploying any identity connectors, shutdown your IDM web app. If the instructions for your bundle indicate that there are no 3rd-party dependencies required, then all you need to do is to place the bundle file into the WEB-INF/bundles folder. If the instructions for your bundle indicate that there are one or more 3rd-party dependencies, then it will be necessary to first download the 3rd-party jars as directed by the bundle's instructions. Once you have downloaded the required jars, you then need to add them to the lib directory within the bundle. You can use the jar command to update your bundle. Once you have updated your bundle with the necessary 3rd-party jar(s), then place the updated bundle jar into the WEB-INF/bundles folder of your IDM web app.

Restart your IDM web app. The connector bundle will now be recognized by IDM and can be used for provisioning.

Which Connectors are currently deployed?

To see which Identity Connectors your IDM instance currently recognizes, follow these steps:

  • Login to the IDM Administrative UI.
  • Select Resources->Resource Type Actions->Configure Managed Resources
  • The section labeled Resource Connectors will list all of the Identity Connectors visible to your IDM web app.

Provisioning to an Identity Connector

After you have a connector successfully deployed, you can create Resources that use the connector as follows:

  • Login to the IDM Administrative UI.
  • Select Resources->Resource Type Actions->New Resource
  • Select the name of the desired connector from the Resource Adapter Type dropdown.
  • If there is more than one version of the connector, you will need to select the desired version using the Version pulldown. If only one version is available, that Version will be pre-selected.
  • If more than one connector host has that version of the connector, you will need to select the desired connector host ("LOCAL" refers to the locally deployed Identity Connectors). If the connector/version is only supported on a single connector host, that Connector Host will be pre-selected.
  • Click the New button.
  • You are now in the standard wizard used to create Resources.

Once you have saved your new Resource, you can use it just like a Resource that was created using a resource adapter.

Using Connectors in an External Gateway

First, be sure that you have started a .NET connector host on your desired machine. Write down the port and authKey that you have configured the remote connector host to use.

Then, using IDM Admin UI, define a corresponding Connector Server entry using the page at Configure->Connector Servers. If IDM can successfully communicate with your connector host, the Status column will show "Available". You can now use connectors that are deployed under that remote connector host.

NOTE: You must login as an administrator with "Resource Administrator" capability in order to see the Configure->Connector Servers page. More information on Connector Servers can be found here.
